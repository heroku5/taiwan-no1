三個雲端網站同步部署流程:

  1. 主要倉儲位於: https://github.com/heroku5/taiwan-no1
  2. 透過 Github Pages 設定完成: https://heroku5.github.io/taiwan-no1
  3. 透過 Heroku CLI 將近端倉儲在 Heroku 登入狀態下以 heroku git:remote -a taiwan-no1 設定 git remote add 後, 再透過 git push heroku 將資料同步至 Heroku 雲端主機 https://taiwan-no1.herokuapp.com
  4. RedHad Openshift 部分則使用 Topology, 點選 php app 後 Start build, 就能從 https://github.com/heroku5/taiwan-no1.git 取得倉儲內容後完成 build , 透過兩段網站跳轉, 從 index.php 跳至 index.html 再跳轉至靜態網站 content/index.html, 完成 http://taiwan-no-1-git-heroku5-dev.apps.sandbox-m2.ll9k.p1.openshiftapps.com/content/index.html

參賽題目

For more Energy Save - Formosa Energy Save - 攏係為著省能源

專案執行流程

執行流程:

  1. 以機械按鈕式及手機遠端模式自動根據各實驗室排課時段進行監控
  2. 從學校教務主機取出機械設計工程系綜一館個實習實驗室的排課時間表
  3. 根據時間表配置冷氣與照明開關的電源開關控制模組
  4. 從手機或瀏覽器對各控制模組進行監控

相關技術:

  1. 綜一館歷年用電統計分析
  2. 以 AJAX POST 從學校教務主機取出各實驗室的排課時間表
  3. 開關控制模組機構設計與製作
  4. Arduino or Raspberry Pi based 機電資整合開關控制模組製作
  5. Rasberry Pi 上的 Python Flask 網際伺服器程式配置
  6. Flutter based Android 與 IOS based App 製作
